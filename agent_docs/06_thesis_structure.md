## Общая структура (7 глав)

### Глава 1: Введение и мотивация (~5 стр.)

**Что написать:**
- Hi-C технология: принцип, что измеряет, зачем нужна
- TAD: что это, биологическая роль (регуляция генов, болезни)
- Проблема: нейросети для детекции TAD (deepTAD и аналоги) требуют размеченных данных
- Мотивация работы: существующие методы разметки плохи → нужен лучший пайплайн

**Ключевые цитаты для поиска:**
- Lieberman-Aiden et al. 2009 (Hi-C)
- Dixon et al. 2012 (TAD открытие)
- Rao et al. 2014 (GM12878, наш датасет)
- Schmitt et al. 2016 (TAD и болезни)

---

### Глава 2: Обзор существующих методов (~15 стр.)

**Что написать:**
- Классификация алгоритмов по механизму (из `01_algorithms.md` §Comparison table)
- Описание каждого из 7 алгоритмов: идея, формула, параметры
- deepTAD: архитектура нейросети + метод сбора датасета (консенсус алгоритмов)
- Критика deepTAD подхода: консенсус как слабое место

**Ключевые статьи:**
- Filippova 2014 (Armatus), Shin 2016 (TopDom), Dixon 2012 (DI+HMM)
- An 2019 (OnTAD), Zheng 2024 (scKTLD)
- deepTAD статья — найти и изучить метод разметки

---

### Глава 3: Критика консенсусного подхода (~10+ стр.) ← ЦЕНТРАЛЬНЫЙ АРГУМЕНТ

**Что показать:**
1. Алгоритмы дают кардинально разные результаты (визуализации, Jaccard matrix)
2. Консенсус размывает биологический сигнал
3. CTCF-валидация: текущие границы не имеют чёткого пика → не биологически информированы
4. Проблема гиперпараметров: одинаковый диапазон для разных хромосом неправомерен

**Визуализации (минимум):**
- Hi-C heatmap с overlay границ всех алгоритмов (chr21 @ 25kb)
- Heatmap попарных Jaccard между алгоритмами
- CTCF enrichment profile для каждого алгоритма (сейчас плохой → аргумент)
- Распределение размеров TAD по алгоритмам (boxplot)
- Пример: где алгоритмы сильно расходятся на конкретном локусе

---

### Глава 4: Механика алгоритмов — почему они дают разные результаты (~8 стр.)

**Что объяснить:**
- Каждый алгоритм оптимизирует разное свойство матрицы
- Insulation-based (TopDom, coiTAD) vs DP (Armatus, OnTAD) vs Spectral (scKTLD)
- Влияние разрешения: при 25kb границы точнее, но шума больше
- Влияние нормализации: RAW vs KR-normalized → разные границы
- Почему нет одного "правильного" ответа без биологической валидации

---

### Глава 5: Предложенный пайплайн (~12 стр.)

**Что описать:**
- Постановка задачи: создать биологически информированный датасет
- Шаг 1: Запуск алгоритмов (наш пайплайн)
- Шаг 2: Консенсус по TAD (пересечения, а не отдельные границы)
- Шаг 3: Статистические методы (Фурье, спектральный) для уточнения границ
- Шаг 4: Валидация через CTCF профиль
- Шаг 5: Отбор финального датасета

---

### Глава 6: Результаты (~10 стр.) ← MAIN RESULT

**Что показать:**
- CTCF профиль для нового датасета vs консенсуса → явный пик
- Количественное сравнение: std профиля, высота пика, процент границ с CTCF
- Примеры локусов: где новый метод лучше
- Сравнение с Arrowhead (эталон Rao 2014)

---

### Глава 7: Заключение (~2 стр.)

- Итог: предложен пайплайн с биологически информированной валидацией
- Вклад: опубликован датасет Hi-C разметки с CTCF-валидацией
- Следующий шаг: обучение нейросети на полученном датасете
